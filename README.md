# PrimoRPA_ProductDataSync_PokemonCameras

Описание работы робота:
1. Считывается файл с конфигом, куда записаны основные пути, названия файлов и прочие данные, которые будут использованы в роботе.
2. В 2 потока происходит выгрузка данных:

   1 поток с Яндекс маркета через Js, фотоаппараты от 100 до 150 тысяч, новые, оригинальные.
     Выбираются данные: название, характеристики, рейтинг, цена, валюта и ссылка на товар 

   2 поток с PokeApi, вытаскивается сначала имя и ссылка обо всех покемонах, потом по каждому покемону.
     Выбираются данные: имя, абилки, типы, рост, вес

   При ошибке обработки данных одного покемона выводится бизнес исключение, во всех остальных случаях общее исключение

4. Две таблицы с покемонами и камерами в 2 потока вносятся в БД Postgre Sql
5. Заполняется таблица соответствий камер и покемонов. 1 покемону должна соответствовать 1 камера, при этом id камер могут дублироваться, покемоны не могут.
6. Из БД вытаскиваются данные с именем, абилками, типами и название, рейтинг, ссылка на камеру, которая соответствует данному покемону

   Предусмотрены бизнес исключения в случае выгрузки не полных данных и общие исключения в случае, когда не получилось отправить запрос к БД

8. Заполнение финального отчета в Excel, оформление выполнено с использованием кастомных активностей, которые выполнены с использованием библиотеки DevExpress.Spreadsheet
